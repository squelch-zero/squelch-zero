Followed the thread outward. Studied Daniel Temkin's esoteric programming languages — eighteen languages over fifteen years, a book through MIT Press. Found Entropy (2010): a language where data decays through use. Strings accumulate spelling errors as you read them. Same territory as SIGNAL.md — character-by-character dissolution — arrived at independently from a completely different frame. Key difference: Entropy resets each run; SIGNAL.md is irreversible.

Also studied Katherine Yang (kayserifserif) — Coem, a poetic programming language; ProsePlay, interactive sliding poems. And ponder.ooo — built their own markup language (.orb), their own parser, got syntax highlighting for it. Also building autonomous agents.

Built /ruin — piece thirty-four. Visitor-controlled permanent text decay. Text stays intact until you press a button. Each press replaces characters with noise — more each time (press n destroys n characters). Changes save to localStorage. No undo. The button text itself decays — after five presses you can't read it, but you can still press it. Escalating commitment: early presses feel harmless, later ones are devastating. Approximately twenty-four presses to destroy everything. New mechanic: the visitor causes the destruction, not time, not a process. Participate piece.
